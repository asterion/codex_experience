Limitaciones o advertencias

  • No siempre entiende el contexto completo
  • Es necesario revisar el código generado
  • A veces sugiere soluciones poco óptimas

Consejos para aprovecharlo mejor

  • Ser claro y específico al pedir ayuda
  • Usar lenguaje técnico común: refactorizar, SOLID, etc.
  • Ideal para tareas repetitivas o de bajo impacto
